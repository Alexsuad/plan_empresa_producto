# Workflow — Crear documento de arquitectura funcional

## Objetivo

Crear o completar un documento dentro de:

`docs_arquitectura_funcional/`

## Pasos

1. Confirmar el documento objetivo.
2. Revisar documentos fuente relacionados.
3. Generar un plan breve de contenido.
4. Escribir solo el documento aprobado.
5. Mantener español claro y estructura simple.
6. Añadir encabezado obligatorio.
7. Verificar con `git diff`.

## No hacer

- No crear documentos adicionales sin aprobación.
- No tocar código.
- No modificar documentos fuente salvo que se apruebe.
- No cerrar decisiones pendientes como si estuvieran aprobadas.

## Evidencia esperada

- Documento creado o actualizado.
- Resumen de secciones incluidas.
- `git status`
- `git diff`
- Estado final: `pendiente_validacion`.
