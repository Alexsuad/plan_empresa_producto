# Mapa Transversal: Anexos, Investigaciones y Validaciones

Referencia de documentos externos, fuentes de investigación y resultados de validación.

---

# Revisión transversal de la guía

Cada punto se analiza bajo los siguientes criterios:

1. **Qué preguntas se repiten**

   Algunas preguntas aparecen en Idea, Mercado, Marketing, Operaciones y Finanzas. No está mal, pero hay que decidir si se mantienen como refuerzo o si se consolidan.

2. **Qué preguntas alimentan cada anexo**

   Por ejemplo:

   | Herramienta / Anexo        | Se alimenta principalmente desde |
   | -------------------------- | -------------------------------- |
   | PESTEL                     | 3.1 Análisis externo             |
   | Cliente ideal              | 2, 3.2 y 6.1                     |
   | Propuesta de valor         | 2, 3.2, 4 y 6.1                  |
   | Benchmark                  | 2, 3.2 y 6.1                     |
   | DAFO                       | 3.1, 3.2 y 3.3                   |
   | CAME                       | 4                                |
   | Curva de valor             | 3.2 y 6.1                        |
   | Matriz de riesgos          | 6.2, 6.3, 6.4, 6.5, 7 y 8        |
   | Tablas financieras         | 6.1, 6.2, 6.3, 6.4 y 6.5         |
   | Calendario de implantación | 5, 6 y 7                         |

3. **Qué debe investigar el emprendedor en cada punto**

   La investigación comprende los siguientes aspectos: competencia, mercado actual, referentes, océano azul, saturación, precios, reseñas, ayudas, evolución del sector, entrevistas, prensa, datos públicos, etc.

4. **Qué preguntas son para entrevista humana y cuáles para sistema agéntico**

   Algunas preguntas son buenas para que el emprendedor responda directamente. Otras son mejores para que el sistema las investigue.

5. **Qué preguntas bloquean avance**

   Por ejemplo:

   * si no sabe qué vende, no puede avanzar a marketing;
   * si no tiene cliente definido, no puede avanzar a ventas;
   * si no hay costes, no puede avanzar a viabilidad;
   * si hay permisos críticos no revisados, no puede recomendarse GO.

# Investigaciones profundas por punto del Plan de Empresa

Porque cada apartado no necesita el mismo tipo de investigación.

Por ejemplo, no se investiga igual:

* el **equipo promotor**;
* la **idea**;
* el **mercado**;
* la **competencia**;
* los **precios**;
* los **permisos legales**;
* las **subvenciones**;
* los **costes financieros**;
* los **riesgos**.

## Dónde encaja esta capa

La jerarquía de capas se define de la siguiente manera:

| Capa                        | Función                                                              |
| --------------------------- | -------------------------------------------------------------------- |
| **Guía humana**             | Explica al emprendedor qué debe pensar y responder.                  |
| **Mapa de anexos**          | Indica qué respuestas alimentan Canvas, DAFO, PESTEL, CAME, etc.     |
| **Mapa de investigación**   | Define qué debe investigar el sistema en cada punto.                 |
| **Mapa de validaciones**    | Define qué respuestas bloquean o permiten avanzar.                   |
| **Sistema funcional posterior** | Usa todo lo anterior para preguntar, investigar, redactar y auditar. |

## Ejemplo de cómo debe verse

Para cada punto del plan se define la siguiente ficha técnica:

```text
Punto del plan:
Qué debe responder el emprendedor:
Qué debe investigar el sistema:
Tipo de investigación:
Fuentes recomendadas:
Profundidad:
Resultado esperado:
Validación mínima:
```

## Ejemplo aplicado

### 3.2 Estudio del mercado

**Qué responde el emprendedor:**

* quién cree que es su cliente;
* qué zona quiere cubrir;
* qué negocios lo inspiran;
* qué precios imagina;
* qué problemas cree que tiene el cliente.

**Qué debe investigar el sistema:**

* tamaño aproximado del mercado;
* competidores reales;
* precios visibles;
* reseñas de clientes;
* señales de saturación;
* huecos de mercado;
* tendencias;
* datos públicos;
* negocios referentes;
* canales de venta usados por competidores.

**Tipo de investigación:**

* investigación rápida;
* investigación profunda;
* análisis de competencia;
* análisis de reseñas;
* benchmark;
* búsqueda de fuentes públicas.

**Fuentes posibles:**

* Google Maps;
* webs de competidores;
* redes sociales;
* INE;
* ayuntamientos;
* Gobierno de Aragón;
* cámaras de comercio;
* asociaciones sectoriales;
* prensa;
* marketplaces;
* informes sectoriales;
* entrevistas a clientes.

**Resultado esperado:**

* listado de competidores;
* precios comparativos;
* mapa de oportunidades;
* señales de saturación;
* huecos detectados;
* conclusiones para el plan.

## Niveles de investigación

Se establecen los siguientes niveles de profundidad de investigación:

| Nivel                      | Cuándo se usa                                                       | Resultado                                                            |
| -------------------------- | ------------------------------------------------------------------- | -------------------------------------------------------------------- |
| **Investigación ligera**   | Para una primera idea o MVP del plan                                | Datos básicos, competidores visibles, primeras fuentes.              |
| **Investigación media**    | Para un plan presentable ante programa de emprendimiento            | Competencia, precios, cliente, tendencias, fuentes citables.         |
| **Investigación profunda** | Para banco, subvención, inversor o decisión de inversión importante | Fuentes sólidas, benchmark, reseñas, evolución, riesgos, escenarios. |

## Ejemplo por puntos principales

| Punto                      | Investigación necesaria                                                                                            |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **1. Equipo promotor**     | Validar experiencia, perfiles públicos, red de apoyo, colaboradores clave si aplica.                               |
| **2. Idea**                | Buscar referentes, negocios similares, modelos existentes, posibles nombres/marca, primeras señales de viabilidad. |
| **3.1 Análisis externo**   | PESTEL, tendencias, normativa general, situación sectorial, ayudas públicas, cambios sociales o tecnológicos.      |
| **3.2 Mercado**            | Mercado objetivo, tamaño, competencia, precios, reseñas, saturación, océano azul, demanda, canales.                |
| **3.3 Análisis interno**   | No requiere tanta investigación externa; más bien validación de recursos, presupuestos y capacidades.              |
| **4. DAFO/CAME**           | No investiga desde cero; usa lo encontrado antes para clasificar y priorizar.                                      |
| **5. Objetivos**           | Contrastar si los objetivos son realistas frente a mercado, capacidad y finanzas.                                  |
| **6.1 Marketing y ventas** | Canales de competidores, mensajes, precios, promociones, embudo, referencias de comunicación.                      |
| **6.2 Operaciones**        | Proveedores, costes, herramientas, tiempos, logística, equipamiento, procesos comparables.                         |
| **6.3 RRHH**               | Convenios, salarios, perfiles, costes laborales, disponibilidad de talento.                                        |
| **6.4 Jurídico-fiscal**    | Forma jurídica, licencias, marca, protección de datos, normativa sectorial, ayudas.                                |
| **6.5 Financiero**         | Costes reales, precios de mercado, inversión inicial, cuotas, alquileres, proveedores, escenarios.                 |
| **7. Implantación**        | Plazos reales de trámites, licencias, proveedores, desarrollo, reformas, apertura o lanzamiento.                   |
| **8. Viabilidad**          | No investiga desde cero; cruza todos los datos y decide Go/Pivot/No-Go.                                            |

## Diseño transversal de la guía: anexos, investigaciones profundas, validaciones y trazabilidad

Bloques a desarrollar:

1. **Anexos que debe producir el sistema.**
2. **Qué preguntas alimentan cada anexo.**
3. **Qué debe responder el emprendedor.**
4. **Qué debe investigar la IA en cada punto.**
5. **Nivel de investigación recomendado: ligera, media o profunda.**
6. **Fuentes recomendadas por tipo de investigación.**
7. **Qué evidencias debe guardar el sistema.**
8. **Qué validaciones bloquean el avance.**
9. **Cómo se conecta todo con el sistema funcional posterior.**
